Descubre las vulnerabilidades, aprende a prevenirlas y comprende sus consecuencias.

1. Conceptos Fundamentales

Vulnerabilidad (Debilidad)

Un defecto en un sistema, aplicación o protocolo que un atacante puede explotar.

Ejemplo: Un código que no revisa el tipo de dato que introduce un usuario.

Amenaza / Ataque (Acción)

Un riesgo potencial de daño. El ataque es la acción de un actor malicioso para explotar esa vulnerabilidad.

Ejemplo: Un atacante realiza una Inyección SQL.

2. Principales Amenazas y Vectores de Ataque

Conocer las tácticas más utilizadas es esencial para la defensa.

Phishing e Ingeniería Social

Técnicas para manipular a las personas y obtener información sensible suplantando una identidad de confianza mediante correos, llamadas o mensajes fraudulentos.

Inyección SQL (SQLi)

Inserción de código SQL malicioso en campos de entrada para manipular bases de datos, robar, modificar o eliminar información sensible.

Cross-Site Scripting (XSS)

Inyección de scripts maliciosos en páginas web que se ejecutan en el navegador de otros usuarios, permitiendo robo de sesiones y datos.

Cross-Site Request Forgery (CSRF)

Engaña al navegador del usuario para ejecutar acciones no autorizadas en sitios donde está autenticado.

Malware y Ransomware

Software malicioso que infecta sistemas para robar datos, espiar actividades o cifrar archivos exigiendo rescate.

Ataques DDoS

Saturación de servidores con tráfico masivo para interrumpir servicios y dejarlos inaccesibles.

Man-in-the-Middle (MitM)

Intercepción de comunicaciones entre dos partes para espiar o manipular datos en tránsito.

Fuerza Bruta y Ataques de Diccionario

Intentos automatizados de adivinar contraseñas probando múltiples combinaciones o palabras comunes.

Credential Stuffing

Uso de credenciales robadas de otras brechas para acceder a múltiples cuentas donde el usuario reutiliza contraseñas.

Path Traversal / Directory Traversal

Acceso no autorizado a archivos y directorios fuera del directorio raíz de la aplicación web.

Inyección de Comandos

Ejecución de comandos arbitrarios del sistema operativo a través de vulnerabilidades en aplicaciones web.

Inclusión de Archivos (RFI/LFI)

Inclusión de archivos maliciosos externos o locales para ejecutar código no autorizado en el servidor.

XXE (XML External Entity)

Explotación de procesadores XML mal configurados para acceder a archivos locales o ejecutar ataques SSRF.

SSRF (Server-Side Request Forgery)

Forzar al servidor a realizar peticiones a recursos internos o externos no autorizados.

Carga de Archivos Maliciosos

Subida de archivos que contienen código malicioso explotando validaciones débiles.

Configuraciones Inseguras

Errores de configuración en servidores, bases de datos o aplicaciones que exponen información sensible.

Autenticación y Autorización Débil

Fallas en mecanismos de autenticación que permiten acceso no autorizado a recursos protegidos.

Exposición de Datos Sensibles

Filtración de información confidencial por falta de cifrado, logs expuestos o respuestas de error verbosas.

Deserialización Insegura

Manipulación de objetos serializados para ejecutar código arbitrario o modificar lógica de aplicación.

Uso de Componentes Vulnerables

Explotación de vulnerabilidades conocidas en librerías, frameworks o dependencias desactualizadas.

Logging & Monitoring Insuficiente

Falta de registros de actividad que impide detectar y responder a incidentes de seguridad a tiempo.

API Abuse

Explotación de APIs mal protegidas para acceder a funcionalidades o datos sin autorización adecuada.

Session Hijacking

Robo de tokens de sesión para suplantar la identidad de usuarios legítimos.

Clickjacking

Engañar a usuarios para que hagan clic en elementos ocultos realizando acciones no intencionadas.

Drive-by Download

Descarga automática de malware al visitar sitios web comprometidos sin interacción del usuario.

Insider Threats

Amenazas internas de empleados o personas con acceso legítimo que abusan de sus privilegios.

Broken Access Control

Fallas en controles de acceso que permiten a usuarios acceder a recursos o funciones no autorizadas.

Business Logic Vulnerabilities

Explotación de fallas en la lógica de negocio de la aplicación para realizar acciones no previstas.

3. Consecuencias de la Materialización del Riesgo

El impacto de una brecha de seguridad se mide en múltiples niveles.

Pérdidas Económicas (Muy altas! (,,>﹏<,,)) Costos de recuperación y multas.
Daño Reputacional Pérdida de confianza.
Interrupción Operacional Caída del servicio.

4. Casos Importantes de Brechas de Seguridad

Ejemplos recientes que demuestran la magnitud de las amenazas.

Ataque a Aerolínea

Robo masivo de datos de clientes y credenciales.

Brecha en la Cadena de Suministro

Compromiso de un proveedor, afectando a múltiples empresas cliente.

5. Estrategias Clave de Prevención

La defensa debe ser multicapa y constante.

Gestión de Parches

Mantener todo el software actualizado.

Validar Entradas de Usuario

Nunca confiar en los datos de entrada para prevenir inyecciones.

Configuración Segura

Asegurar que las configuraciones de los sistemas sean restrictivas.

6. Factores de Riesgo: Técnicos vs. Humanos

Factores Técnicos (Diapositiva 8)

  • Uso de consultas parametrizadas.
  • Implementar Autenticación Multifactor (MFA).
  • Pruebas de seguridad regulares.

Factor Humano (Diapositiva 9)

  • Formación Continua.
  • Buena Gestión de Contraseñas.
  • Fomentar la Cultura de Seguridad.

7. Conclusión: El Triángulo de la Seguridad

La seguridad web se fundamenta en la responsabilidad de todos.

Prevención Activa

Mitigar vulnerabilidades.

Detección Temprana

Monitorear y responder a las amenazas.

Recuperación

Planes de respaldo y restauración.

Responsabilidad

Compromiso individual.
