Ciberseguridad Web: Amenazas y Defensa

Descubre las vulnerabilidades, aprende a prevenirlas y comprende sus consecuencias.

1. Conceptos Fundamentales

Vulnerabilidad (Debilidad)

Un **defecto** en un sistema, aplicación o protocolo que un atacante puede explotar.

**Ejemplo:** Un código que no revisa el tipo de dato que introduce un usuario.

Amenaza / Ataque (Acción)

Un **riesgo** potencial de daño. El ataque es la acción de un actor malicioso para explotar esa vulnerabilidad.

**Ejemplo:** Un atacante realiza una Inyección SQL.

2. Principales Amenazas y Vectores de Ataque

Conocer las tácticas más utilizadas es esencial para la defensa.

Phishing e Ingeniería Social

Técnicas para manipular a las personas y obtener información sensible suplantando una identidad de confianza.

Inyección SQL y XSS

Inserción de código malicioso para robar datos, comprometer la base de datos o secuestrar sesiones.

3. Consecuencias de la Materialización del Riesgo

El impacto de una brecha de seguridad se mide en múltiples niveles.

Pérdidas Económicas ($10.5T) Costos de recuperación y multas.
Daño Reputacional Pérdida de confianza.
Interrupción Operacional Caída del servicio.

4. Casos Importantes de Brechas de Seguridad

Ejemplos recientes que demuestran la magnitud de las amenazas.

Ataque a Aerolínea

Robo masivo de datos de clientes y credenciales.

Brecha en la Cadena de Suministro

Compromiso de un proveedor, afectando a múltiples empresas cliente.

5. Estrategias Clave de Prevención

La defensa debe ser multicapa y constante.

Gestión de Parches

Mantener todo el software actualizado.

Validar Entradas de Usuario

Nunca confiar en los datos de entrada para prevenir inyecciones.

Configuración Segura

Asegurar que las configuraciones de los sistemas sean restrictivas.

6. Factores de Riesgo: Técnicos vs. Humanos

Factores Técnicos (Diapositiva 8)

  • Uso de consultas parametrizadas.
  • Implementar Autenticación Multifactor (MFA).
  • Pruebas de seguridad regulares.

Factor Humano (Diapositiva 9)

  • Formación Continua.
  • Buena Gestión de Contraseñas.
  • Fomentar la Cultura de Seguridad.

7. Conclusión: El Triángulo de la Seguridad

La seguridad web se fundamenta en la responsabilidad de todos.

Prevención Activa

Mitigar vulnerabilidades.

Detección Temprana

Monitorear y responder a las amenazas.

Recuperación

Planes de respaldo y restauración.

Responsabilidad

Compromiso individual.
